# Siro API Framework v0.6.1 Verification

This checklist verifies all mandatory fixes delivered in v0.6.1.

## 0) Prep

1. Run migrations:

```bash
php siro migrate
```

2. Confirm migration state:

```bash
php siro migrate:status
```

Expected:
- `20260427110100_create_users_table.php` = applied
- `20260427143000_add_token_version_to_users_table.php` = applied

---

## 1) JWT revoke + invalid token rejection

1. Register or login to get token **A**.
2. Call protected endpoint:

```bash
curl -i -H "Authorization: Bearer <TOKEN_A>" http://localhost:8000/api/auth/me
```

Expected: `200`.

3. Revoke via logout:

```bash
curl -i -X POST -H "Authorization: Bearer <TOKEN_A>" http://localhost:8000/api/auth/logout
```

Expected: `200` + revoke success message.

4. Reuse old token **A**:

```bash
curl -i -H "Authorization: Bearer <TOKEN_A>" http://localhost:8000/api/auth/me
```

Expected: `401` (token version mismatch / revoked token).

5. Tamper token or use invalid signature.

Expected: `401` (invalid/expired token).

---

## 2) Cache invalidation scope

1. Hit cached endpoints multiple times to warm query cache.
2. Mutate `users` data (POST/PUT/DELETE users).
3. Validate only `qb:users:*` keys are invalidated.

Expected:
- No global `qb:` flush.
- Unrelated table cache (e.g. `qb:orders:*`) remains.

---

## 3) Transactions

Run a small integration check in app code:

```php
Database::transaction(function () {
    DB::table('users')->insert([...]);
    DB::table('orders')->insert([...]);
});
```

Expected: both persisted.

Rollback test:

```php
try {
    Database::transaction(function () {
        DB::table('users')->insert([...]);
        throw new RuntimeException('force rollback');
    });
} catch (Throwable) {
}
```

Expected: no partial write from failed transaction.

---

## 4) Rate limit concurrency / TTL safety

Send concurrent requests to a throttled route (same IP + same route).

Expected:
- Counter increments correctly.
- TTL remains set (no `-1` no-expiry state).
- 429 returned when threshold exceeded.

---

## 5) Migration CLI usability

Status:

```bash
php siro migrate:status
```

Expected: accurate applied vs pending list.

Rollback one step:

```bash
php siro migrate:rollback --step=1
```

Expected: exactly latest migration rolled back.

Rollback N steps:

```bash
php siro migrate:rollback --step=2
```

Expected: latest 2 applied migrations rolled back in reverse order.
